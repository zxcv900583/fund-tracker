## v1.18 更新摘要（2026-06-14）— 賣出計入損益（平均成本法 / 已實現）

- **平均成本法會計**：新增 `positionOf(h)`，依日期重播買入與賣出（同日視賣出晚於買入），算出**目前持有股數（已扣賣出）**、**剩餘成本基礎**、**已實現損益**與**累計投入**。賣出時以當時平均成本認列已實現損益（賣出價金 − 平均成本 × 賣出量 − 手續費）。`totUnits`/`totCost`/`avgNav` 改以此為準，新增 `realizedPL(h)`。
- **總損益定義**：每檔 `總損益 = 未實現（持有市值 − 剩餘成本）＋ 已實現 ＋ 配息`；報酬率改以**累計投入**為分母（避免部分/全部賣出後分母失真）。
- **XIRR 計入賣出**：`fundXirr` 與組合 `renderHeader` 的現金流加入賣出價金（正流），使年化報酬涵蓋已實現部分。
- **組合走勢圖**：`buildPortfolioSeries` 重播賣出 → 持有市值與剩餘成本隨賣出日下降，新增 `realArr` 累計已實現，tooltip 顯示「總損益＝未實現＋配息＋已實現」。
- **管理彈窗 / 賣出表**：`#mngXirr` 顯示總損益並拆「未實現／已實現／配息」；賣出區摘要顯示「累計賣出量＋已實現損益」；買入表「合計」維持**累計買入**（與下方買入列一致）。賣出量超過累計買入時會提示，超出部分不計入損益。
- **相容性**：舊資料無 `sells` → `positionOf` 等同原行為；0 股觀察持倉（v1.17）維持成本/損益 0。

## v1.17 更新摘要（2026-06-14）— 觀察用持倉（0 股 / 0 單位）

- **0 股觀察**：買入記錄的「股數／單位數」可填 0，用來把標的加入清單**純追蹤走勢**而不持有部位。此時成本、現值、含息損益皆為 0，報酬率因成本為 0 顯示「—」，XIRR 不計入該標的；目前價格、當日漲跌、區間漲跌與走勢圖仍正常顯示。
- **驗證放寬**：`savePur`、`confirmStock`（加股票）、`confirmAdd`（加基金）改為允許 0 股 —— 0 股時不要求輸入價格、不寫入成交金額（`amount=null`，成本 0）；有股數時才強制要價格。
- **`rowData` 調整**：現值由 `units>0 ? units*nav : null` 改為 `units*nav`（0 股→現值 0、損益 0），其餘百分比計算本就有 `cost>0` 防護，表格與摘要不會出現 NaN/Infinity。

## v1.16 更新摘要（2026-06-14）— 浮點精度修正＋賣出表單自動帶入

- **修正浮點雜訊**：Yahoo 股價為 32-bit float，例如 `1873.8707275390625` 實為 `1873.87`，自動帶入時直接塞入輸入框會顯示超長尾數。新增 `cleanPrice(v)`（價格 ≥100 取 2 位小數、<100 取 4 位，與表格 `fmtN` 一致）與 `cleanAmount(v)`（金額取 4 位）。輸入框顯示（`openPurEdit`/`openSellEdit`）、自動帶入（`autoFillNavForDate`/`autoFillSellPriceForDate`/`navLookup`）、儲存（`savePur`/賣出儲存）皆套用，數值乾淨且存檔即清理。表格與圖表本就用 `fmtN`，不受影響。
- **賣出表單自動帶入**：賣出記錄編輯區（`#sellEdit`）的「賣出日」選好後自動帶入該日數值（基金=淨值、股票=收盤價，可手動覆寫），開啟新增亦依預設日期靜默預填。新增 `autoFillSellPriceForDate({silent, cacheOnly})` 並掛 `#eSellDate` 的 `change` 事件；基金查無快取改用 `msHistory` 連 API 補。**此功能僅帶入價格，賣出仍不影響持有股數與損益會計（維持 v1.14 輕量版約束）。**

## v1.15 更新摘要（2026-06-14）— 買入表單自動帶入當日數值

- **選好日期即自動帶入**：管理彈窗「新增買入記錄」改為選好「買入日期」後自動帶入該日數值——**基金**帶當日淨值、**股票**帶當日收盤價，兩者皆可手動覆寫（股票實際成交價可能異於收盤，故提示「可修改」）。開啟表單時亦依預設日期（今天）先以快取靜默預填。
- **快取優先、API 後備**：優先讀本地 `navHistory` 快取（瞬間、免連網），基金查無快取才自動連 Morningstar 補；股票無快取則提示先點「⟳ 更新」抓歷史。新增 `navOnOrBefore(list, date, maxBack)`（取該日或之前最近交易日）與 `autoFillNavForDate({silent, cacheOnly})`，並在 `#eDate` 的 `change` 事件觸發。
- **不覆寫既有值**：編輯既有買入記錄時不會被預填覆寫（僅在「新增」且使用者實際更動日期時帶入）。

## v1.14 更新摘要（2026-06-14）— 走勢圖買賣標記（輕量版）

## Worker 安全設計（v8）

> v8：報價 `normalizeQuote` 另帶出 `session:{start,end}`（Yahoo `currentTradingPeriod.regular`），供前端顯示各市場休市標籤。

> 原始碼為乾淨 ES module（非 esbuild 打包產物），可直接 `wrangler dev` / `wrangler deploy`；部署位置 `https://fund-tracker-proxy.fund-tracker-proxy.workers.dev`。

| 機制 | 說明 |
|------|------|
| Origin 白名單 | 瀏覽器跨站請求僅允許正式站 `https://zxcv900583.github.io` 與本機開發來源；無 `Origin`（curl／`file://`）放行，其餘回 **403**。綁網站網域而非使用者，故任何訪客從本站操作皆放行，僅阻擋他站盜用為免費 API。 |
| 速率限制 | 每 IP 一般 API 120 次/分、`deep health` 5 次/分（isolate 記憶體版，擋單點濫用；全域防護建議另加 Cloudflare WAF rate-limiting 規則）。 |

| chart `fresh` | `/api/yahoo/chart?fresh=1` 強制清快取（回 `no-store`），供「強制更新」真正取得最新資料。 |
| 報價對帳 | `normalizeQuote` 盤中以 meta 即時報價、收盤後退回 K 線陣列；前收一律用 `previousClose` 或 K 線前一日，不使用 `chartPreviousClose`（除權息調整會使當日漲跌計算失真）。 |

## 1. 專案概述

### 1.1 背景

目前透過基富通證券網站管理基金投資組合，**必須登入才能查看淨值與損益**。本專案目標是打造一個**免登入、免安裝、手機可用**的個人基金追蹤工具，行為類似桌面版券商軟體（參考：Internet Quotes Assistant 截圖），但以 PWA 網頁形式呈現。

### 1.2 核心目標

| 目標 | 說明 |
|------|------|
| 免登入 | 不需任何帳號，開啟即用 |
| 免安裝 | 純網頁 PWA，手機可加入主畫面 |
| 本地持久化 | 投資組合資料存於瀏覽器 localStorage |
| 自動追蹤淨值 | 呼叫公開 API 取得最新每日淨值 |
| 損益計算 | 依買入記錄計算報酬率、損益金額 |
| 歷史折線圖 | 多種時間範圍切換的淨值走勢圖 |

### 1.3 參考 UI

參考截圖為 **基富通_翔.IQA - Internet Quotes Assistant**，其核心版面如下：

- 上方：投資組合**列表表格**（每行一支基金），顯示最新淨值、損益%、持有單位數、投資金額等
- 下方：點選任一基金後顯示該基金的**歷史淨值百分比折線圖**（以買入均價為 100% 基準）
- 底部狀態列：顯示總投資金額與總損益

## 2. 技術架構

### 2.1 形式

**單檔 PWA + 無狀態 Cloudflare Worker**

- 單一 HTML 檔案（或 Vite/React 打包）
- 持倉資料不傳至後端；Worker 僅代理固定市場／基金資料路由並提供 KV 備援快取
- 手機瀏覽器可「加入主畫面」離線使用（Service Worker）
- 淨值資料仍需網路才能更新

### 2.2 技術選型建議

| 層次 | 建議技術 | 備註 |
|------|----------|------|
| 框架 | React 18 + Vite | 或純 HTML/JS 亦可 |
| 圖表 | Chart.js 或 Recharts | 輕量、手機友善 |
| 樣式 | Tailwind CSS | 快速 RWD |
| 儲存 | localStorage | 無帳號版本 |
| API 呼叫 | fetch() | 瀏覽器原生 |
| PWA | vite-plugin-pwa | Service Worker 自動生成 |
| 搜尋 | TDCC OpenAPI | 見第 3 節 |

### 2.3 跨域問題（CORS）說明

> **重要：** `openapi.tdcc.com.tw` 的 CORS 政策未必允許前端直接呼叫。
> 若遇到 CORS 錯誤，需在中間架設一個**輕量 Proxy**（可用 Cloudflare Worker 或 Vercel Edge Function，免費方案足夠），轉發 TDCC API 請求。

## 3. 資料來源 & API 規格

### 3.1 主要資料源：臺灣集中保管結算所 OpenAPI

**Base URL：** `https://openapi.tdcc.com.tw`  
**文件：** `https://openapi.tdcc.com.tw/tdcc-opendata-api-docs`  
**授權：** 政府資料開放授權條款第 1 版（免費、免 API Key）

#### 3.1.1 國內基金（境內基金）淨值

> TDCC OpenAPI 目前公開的是**期信基金**（期貨信託基金）淨值（endpoint `/v1/opendata/5-4`）。
> 一般境內股票型基金（如 UNI023 統一奔騰）屬於**證券投資信託基金**，TDCC 本身不直接提供單檔查詢 endpoint，需使用以下替代方案。

#### 3.1.2 境外基金淨值

TDCC OpenAPI 提供境外基金淨值查詢，可直接呼叫：

```
GET https://openapi.tdcc.com.tw/v1/opendata/1-1
```

回傳欄位：`資料日期`、`基金代碼`、`基金名稱`、`計價幣別`、`淨值`、`淨值日`

#### 3.1.3 境內基金（一般投信基金）淨值 — 替代方案

由於 TDCC OpenAPI 對境內基金沒有開放單一代碼查詢，採用以下策略：

**方案 A（推薦）：政府資料開放平台**

```
GET https://data.gov.tw/api/v2/rest/datastore/11454?
    filters=基金代碼:UNI023
    &limit=1
    &sort=淨值日 desc
```

> 資料集 ID `11454` 為「國內基金淨值」，由金管會每日更新。

**方案 B：SITCA 投信投顧公會網站爬取**

- URL：`https://www.sitca.org.tw/ROC/Industry/IN2105.aspx`
- 方式：POST 表單，需 proxy 繞過 CORS
- 可取得歷史淨值資料（完整歷史）

**方案 C（備援）：Morningstar 非官方 API**

```
GET https://lt.morningstar.com/api/rest.svc/timeseries_price/
    9vehuxllxs?id=F000009XIM:NTW:TWF
    &currencyId=TWD&idtype=Morningstar&frequency=daily
    &startDate=2025-01-01&endDate=2026-06-10&outputType=JSON
```

> 基富通的基金頁面資料即來自 Morningstar，格式較穩定但非官方授權。

### 3.2 基金搜尋

**方案：呼叫 TDCC 基金清單，前端搜尋**

1. 啟動時一次性載入全部基金清單（TDCC `/v1/opendata/5-1` 或政府開放平台基金基本資料）
2. 前端 fuzzy search（依代碼或名稱）
3. 使用者選擇後記錄至 localStorage

## 6. 功能模組規格

### 6.1 基金搜尋模組

**輸入：** 使用者輸入的字串（代碼 or 中文名稱）  
**流程：**

1. App 啟動時，從 TDCC API 或政府開放平台載入**全部基金清單**，快取至 `fund_tracker_fund_list`
2. 搜尋時對快取清單進行前端 fuzzy match（代碼完全符合優先，名稱關鍵字次之）
3. 若清單過期（> 7 天）則背景更新

**清單來源 API：**

```
GET https://openapi.tdcc.com.tw/v1/opendata/5-1   （期信基金總覽）
GET https://openapi.tdcc.com.tw/v1/opendata/3-1   （境外基金基本資料）
GET https://data.gov.tw/api/v2/rest/datastore/<dataset_id>  （境內基金）
```

### 6.2 淨值更新模組

**觸發時機：**
- App 開啟時（若距上次 > 6 小時）
- 使用者點擊「⟳ 更新」按鈕
- 手動強制重整（下拉刷新，手機）

**流程：**

```
For each fundCode in holdings:
  1. 呼叫 API 取得最新淨值
  2. 比對 localStorage 快取
  3. 若有新資料 → 追加至 navHistory
  4. 更新 lastUpdated timestamp
  5. 若 API 失敗 → 顯示 "使用快取資料" 提示，不清空舊資料
```

**批次限流：** 若持有基金 > 10 支，每 500ms 發一個請求，避免 API Rate Limit

### 6.3 歷史資料累積策略

由於 TDCC 等 API 僅保留近 1 年資料，需在本地累積：

- 每次更新時，將新取得的淨值**合併**至 localStorage 快取（不覆蓋舊資料）
- 最多保留 730 筆（約 2 年），超過則移除最舊的
- 若使用者清除瀏覽器資料，歷史會遺失（此為 localStorage 方案的已知限制）

### 6.4 資料匯出/匯入

- **匯出：** 將全部 localStorage 資料轉為 JSON 檔案下載
- **匯入：** 上傳 JSON 檔案還原資料（換裝置轉移用）
- 位置：設定頁面

## 7. 計算邏輯

### 7.1 加權平均買入成本

```
持有總單位數 = Σ purchases[i].units
總買入成本   = Σ (purchases[i].units × purchases[i].navAtPurchase)
加權平均淨值 = 總買入成本 ÷ 持有總單位數
```

### 7.2 損益計算

```
現值       = 持有總單位數 × 最新淨值
損益金額   = 現值 − 總買入成本
損益率 (%) = (損益金額 ÷ 總買入成本) × 100
```

### 7.3 時間區間報酬率

```
區間開始淨值 = navHistory 中最接近（≥）開始日期的淨值
區間報酬率  = (最新淨值 − 區間開始淨值) ÷ 區間開始淨值 × 100%
```

### 7.4 當日漲跌

```
當日漲跌(金額) = 今日淨值 − 前一營業日淨值
當日漲跌(%)   = 當日漲跌(金額) ÷ 前一營業日淨值 × 100%
```

若今日淨值尚未更新（e.g. 假日），則使用最近兩筆有資料日期計算。

### 7.5 總損益摘要（Header）

```
總投資成本 = Σ 各基金的 總買入成本（統一換算為 TWD）
總現值     = Σ 各基金的 現值（統一換算為 TWD）
總損益金額 = 總現值 − 總投資成本
總報酬率   = 總損益金額 ÷ 總投資成本 × 100%
```

**幣別換算：** USD、EUR 等外幣計價基金，需乘以對應匯率（可呼叫 TDCC 匯率 API 或 Yahoo Finance 匯率）

## 9. 限制與已知問題

| 項目 | 說明 |
|------|------|
| localStorage 上限 | 約 5MB，持有大量基金且長期累積後可能不足，建議定期匯出 |
| CORS 問題 | TDCC API 可能不允許瀏覽器直接存取，需 Proxy 中轉 |
| 換裝置資料遺失 | localStorage 不跨裝置，需手動匯出/匯入 |
| 境內基金 API 不穩定 | 政府開放平台 API 偶爾維護中斷，需有 fallback 機制 |
| 歷史資料上限 | TDCC 等 API 僅保留近 1 年資料，更早的依賴本地累積 |
| 假日無淨值 | 非營業日不更新，屬正常現象，App 需顯示「最近營業日」提示 |
| 外幣匯率 | 外幣基金換算 TWD 總損益需即時匯率，另行呼叫匯率 API |
